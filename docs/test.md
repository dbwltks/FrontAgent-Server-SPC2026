테스트 시나리오
설정: 청소 업체 AI 상담사

1. 지식 검색
   "안녕하세요"

"콜비는 뭐예요?"

"이사 청소 얼마예요?"

"A/S는 어떻게 신청해요?"

2. 예약
   "어떤 서비스가 있나요?"

"화장실 청소 예약하고 싶어요"

정보 수집 순서: 날짜/시간 → 성함 → 연락처 (한 번에 다 묻지 않고 단계별로 진행)

"이번 주 금요일 오후 2시요"

"김민준이요" (성함만 답해도 "맞으신가요?" 재확인과 동시에 즉시 저장됨)

"네"

"010-1234-5678"

예약 불가(휴무일/이미 예약된 시간) 시: "O일은 예약이 불가하다"와 "전체 접수 가능 기간(30일 등)"을 한 문장에 섞어 말하지 않는지 확인. 가능한 시간대는 슬롯을 다 나열하지 않고 "O시부터 O시까지" 식으로 1~2개 구간만 안내하는지 확인.

3. 조회
   "예약 확인하고 싶어요"

"010-1234-5678" → 가장 가까운(다가올) 예약 1건만 자연스러운 한 문장으로 안내 + 나머지 건수 안내. 지난 예약은 기본적으로 제외됨.

"네 자세히 알려주세요"
   - 예약이 3건 이하면 바로 전체를 자연어 문장으로 안내.
   - 4건 이상이면 곧바로 나열하지 않고 "이번 주 걸 보여드릴까요, 다음 주 걸 보여드릴까요?" 식으로 먼저 되물음 → 조건에 맞는 것만 골라 안내.

4. 취소
   "예약 취소하고 싶어요"

"010-1234-5678" → 가장 가까운 예약 1건을 "OO월 OO일 OO시 [서비스] 예약을 취소할까요?"로 자연스럽게 확인 (번호 목록을 먼저 보여주지 않음)

"네" → 그 예약이 바로 취소됨 ("예약이 취소되었습니다")

또는 "아니요, 다른 거요" → 방금 확인한 것만 제외하고 나머지를 "번호. 날짜시간 서비스명" 형식으로 안내 → 번호로 선택하면 그 예약이 취소됨

5. 진행 중 다른 태스크로 전환
   조회/취소/예약 중 아무 단계에서나 "예약할래요"처럼 전혀 다른 요청을 보내면, 진행 중이던 태스크를 정리하고 새 태스크로 바로 전환되는지 확인 (예: 조회 도중 "예약할래요" → 서비스 선택부터 새로 시작).

물음표로 끝나는 요청("예약할 수 있나요?", "취소할 수 있을까요?", "조회할 수 있나요?")도 정상적으로 해당 태스크가 트리거되는지 확인 — 지식검색(FAQ)으로 새지 않아야 함.
